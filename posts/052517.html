What is the .freeze method?

The .freeze method is a way to turn mutable objects into immutable ones. This means that once .freeze is run on an object it can no longer be changed at any point. The .freeze works for any decendant of the Object class, meaning anything used inside Ruby can be "frozen".

via GIPHY

What exactly does this mean?

All objects in Ruby are, by default, mutable which means they can be altered, changed, manipulated, or whatever other word you would prefer to use. THIS INCLUDES CONSTANTS which by definition should be...wait for it...constant or unchanging. Don't believe me? Go ahead open up whichever repl you feel most attached to and run this example yourself.

CONSTANT = "foo"
# => "foo"

CONSTANT << "bar"
# => "foobar"

CONSTANT = "new string"
# => "new string"

As you can see your "constant" can be added to or completely reset to a new value. This is bad. As well educated developers we know that once we set a constant it should be just that, constant.

The other example we can look at is strings. Any time a string literal, an actual string is used somewhere inside your code, is used it has a new object_id or a new location in memory.

"string".object_id
# => 70173781215420

"string".object_id
# => 70173781198020

"string".object_id
# => 70173781149560

This could make your program very slow and inefficent. We have symbols that can fix this for us, but there are some instances where can't or shouldn't be used.

Enter the .freeze

Short and simple .freeze is awesome. Any object you use it on will be completely unchangeable. End of story.

Small side note

Yuhihiro Matsumoto, or Matz, the creator of Ruby, shared on twitter in August of 2015 that in Ruby 3.0 all strings will by default be immutable.
